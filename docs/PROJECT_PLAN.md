\# PROJECT\_PLAN.md — CareerFit AI 프로젝트 기획 초안



\## 1. 문제 정의



대학생들은 취업, 인턴, 공모전, 대외활동을 준비할 때 어떤 경험과 역량을 쌓아야 하는지 판단하기 어렵다.

공고마다 요구하는 기술 스택, 우대사항, 활동 경험이 다르기 때문에 사용자가 직접 여러 공고를 비교하고 분석해야 한다.



CareerFit AI는 사용자의 전공, 보유 스킬, 관심 직무를 바탕으로 취업·공모전 데이터를 검색하고, 사용자의 현재 역량과 목표 사이의 차이를 분석해 맞춤형 포트폴리오 방향을 제안하는 서비스이다.



\## 2. 사용자 정의



\### 대상 사용자



\* 취업, 인턴, 공모전, 대외활동을 준비하는 대학생

\* 자신의 전공과 스킬을 기반으로 어떤 활동을 해야 할지 고민하는 학생

\* 포트폴리오를 어떻게 구성해야 할지 모르는 초보자



\### 사용 시나리오



1\. 사용자가 자신의 전공, 보유 스킬, 관심 직무를 입력한다.

2\. CareerFit AI가 취업·공모전 데이터를 검색한다.

3\. 사용자의 입력과 데이터의 요구 역량을 비교한다.

4\. 부족한 역량, 추천 활동, 포트폴리오 개선 방향을 제안한다.

5\. 어떤 데이터를 기반으로 답변했는지 출처를 함께 표시한다.



\## 3. 데이터 계획



\### 취업 공고 컬럼



\* 회사명

\* 직무명

\* 요구 기술 스택

\* 우대사항

\* 주요 업무

\* 자격 요건

\* 마감일

\* 출처 URL



\### 공모전 컬럼



\* 공모전명

\* 주최 기관

\* 모집 대상

\* 요구 역량

\* 활동 내용

\* 마감일

\* 출처 URL



\### 데이터 출처



\* 강사 제공 목업 CSV 데이터

\* 추후 확장 시 공개 채용 공고 및 공모전 사이트 데이터 활용 가능



\### 데이터 처리 계획



\* Pandas로 CSV 데이터를 읽는다.

\* 결측치와 중복 데이터를 확인한다.

\* 기술 스택 키워드를 표준화한다.

\* SQLite에 정형 데이터를 저장한다.

\* ChromaDB에 검색용 문서 형태로 변환해 저장한다.



\## 4. 기능 목록



\### 백엔드 기능



\* FastAPI 서버 구성

\* `/health` 엔드포인트로 서버 상태 확인

\* `/jobs` 엔드포인트로 목업 데이터 조회

\* `/analyze` 엔드포인트로 사용자 입력 분석

\* Gemini API를 활용한 맞춤형 응답 생성

\* ChromaDB 기반 관련 문서 검색

\* 응답 JSON에 분석 결과와 출처 포함



\### 프론트엔드 기능



\* 사용자 입력 폼 구성

\* 전공, 보유 스킬, 관심 직무 입력

\* 분석 요청 버튼

\* AI 분석 결과 카드 출력

\* 참고한 데이터 출처 카드 출력



\### 문서화 기능



\* README에 문제 정의, 데이터, 구현, 검증 과정 정리

\* CHECKLIST.md로 진행 상황 관리

\* EVAL\_QUESTIONS.md로 자기 평가 질문 정리
