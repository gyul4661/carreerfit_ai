# CareerFit AI UI 디자인 규칙

## 1. 프로젝트 개요

CareerFit AI는 취업·공모전 데이터 기반 맞춤형 AI 포트폴리오 코치 서비스이다.

사용자는 자신의 전공, 보유 스킬, 관심 직무를 입력하고, AI는 관련 공고를 참고하여 역량 분석 결과와 포트폴리오 개선 방향을 제공한다.

UI는 초보자도 쉽게 사용할 수 있도록 단순하고 직관적으로 구성한다.

---

## 2. 컬러 팔레트

- primary: `#3B82F6`  
  - 파란색 계열
  - 신뢰감, 전문성, 기술 서비스 느낌을 표현
  - 주요 버튼, 강조 요소에 사용

- secondary: `#10B981`  
  - 초록색 계열
  - 성장, 추천, 긍정적인 피드백 표현
  - 분석 결과나 긍정 메시지에 사용

- background: `#F8FAFC`  
  - 연한 회색 배경
  - 화면 전체 배경색으로 사용

- text-primary: `#1E293B`  
  - 진한 남색/회색
  - 제목, 주요 텍스트에 사용

- text-muted: `#64748B`  
  - 흐린 회색
  - 설명 문구, 보조 텍스트에 사용

- border: `#E2E8F0`  
  - 연한 회색
  - 카드, 입력창 테두리에 사용

- error: `#EF4444`  
  - 빨간색
  - 오류 메시지에 사용

---

## 3. 타이포그래피

- 제목: `text-2xl font-bold text-slate-800`
- 소제목: `text-lg font-semibold text-slate-700`
- 본문: `text-base text-slate-600`
- 설명: `text-sm text-slate-500`
- 버튼: `text-sm font-medium`

텍스트는 너무 작지 않게 하고, 모바일에서도 읽기 쉽게 구성한다.

---

## 4. 컴포넌트 구조

### App.jsx

- 전체 페이지 레이아웃 담당
- API 요청 상태 관리
- 로딩, 오류, 결과 출력 영역 관리

### InputForm.jsx

- 사용자 입력 담당
- 전공, 보유 스킬, 관심 직무 입력
- 버튼 클릭 시 `/analyze` 요청에 필요한 데이터를 상위 컴포넌트로 전달

### ResultCard.jsx

- AI 분석 답변 출력
- 답변은 줄바꿈이 유지되도록 `whitespace-pre-line` 사용
- 결과 영역은 눈에 잘 띄도록 초록색 계열 테두리 사용 가능

### SourceCard.jsx

- 참고한 공고 출처 목록 출력
- 회사명, 직무명, 필수 스킬을 표시
- 데이터가 없을 경우 안내 문구 출력

---

## 5. 레이아웃 규칙

- 최대 너비: `max-w-2xl mx-auto`
- 화면 배경: `min-h-screen bg-slate-50`
- 카드 스타일: `bg-white rounded-xl shadow-sm border border-slate-200 p-6`
- 섹션 간격: `space-y-4`, `mt-8`
- 입력창 스타일: `w-full border border-slate-300 rounded-lg px-3 py-2 text-sm`
- 버튼 스타일: `w-full bg-blue-500 hover:bg-blue-600 disabled:bg-slate-300 text-white font-medium py-2 px-4 rounded-lg`

---

## 6. UI 작성 원칙

- 한 화면에서 사용자가 무엇을 입력해야 하는지 바로 알 수 있어야 한다.
- 분석 결과와 참고 공고 출처는 분리해서 보여준다.
- 오류 메시지는 사용자가 원인을 이해할 수 있게 작성한다.
- 로딩 중에는 버튼 문구와 화면 메시지로 상태를 알려준다.
- 불필요한 아이콘이나 장식은 최소화한다.
- 텍스트 기반 UI를 우선한다.

---

## 7. 금지 사항

- React 코드에 API Key를 직접 넣지 않는다.
- 백엔드 API 주소 외에 민감한 정보를 프론트에 노출하지 않는다.
- 너무 많은 색상을 사용하지 않는다.
- 전체 코드를 불필요하게 다시 작성하지 않는다.
- 사용자 입력 없이 API 요청이 자동 실행되게 만들지 않는다.
- 출처 데이터가 없는데 임의로 만들어 표시하지 않는다.

---

## 8. 검수 기준

UI를 수정한 뒤 아래 항목을 확인한다.

- 전공, 스킬, 관심 직무 입력창이 정상 표시되는가?
- 입력값이 없을 때 버튼이 비활성화되는가?
- 버튼 클릭 시 `/analyze` API 요청이 실행되는가?
- 로딩 중 문구가 표시되는가?
- 오류 발생 시 오류 메시지가 표시되는가?
- AI 분석 결과가 카드 형태로 표시되는가?
- 참고 공고 출처가 카드 형태로 표시되는가?
- Tailwind CSS 클래스가 지나치게 복잡하지 않은가?
